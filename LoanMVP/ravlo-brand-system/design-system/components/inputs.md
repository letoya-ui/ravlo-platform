Inputs



Inputs are the core interactive elements of the Ravlo interface. They collect information, guide decisions, and help investors move through workflows with clarity and confidence.



Ravlo inputs are designed to feel:

\- clean

\- structured

\- predictable

\- cinematic

\- investor‑grade



1\. Purpose

Inputs allow users to enter, adjust, or refine data. They must reduce friction, prevent confusion, and support fast, accurate decision‑making.



Inputs are used for:

\- deal analysis fields

\- financial inputs

\- property details

\- partner onboarding

\- settings and preferences

\- search and filtering

\- workflow steps



2\. Types of Inputs

Ravlo uses a disciplined set of input types:



Text Input

Used for general text entry such as names, addresses, or notes.



Number Input

Used for financial values, quantities, and calculations. Includes formatting rules.



Select Input

A dropdown used for choosing from a predefined list.



Search Input

A type‑ahead input used for searching properties, partners, or data.



Textarea

Used for longer text entries such as descriptions or comments.



Toggle / Switch

Used for binary choices such as on/off or yes/no.



Checkbox

Used for selecting one or more options.



Radio Button

Used for selecting a single option from a small set.



Date Input

Used for selecting dates with clarity and consistency.



3\. Anatomy

A Ravlo input includes:

\- label text

\- input field

\- optional icon

\- helper text (optional)

\- validation message (optional)

\- spacing based on tokens

\- radius based on radii tokens



4\. Behavior

Inputs must:

\- maintain consistent spacing and alignment

\- use clear, descriptive labels

\- avoid placeholder‑only labeling

\- provide real‑time validation when appropriate

\- support keyboard navigation and screen readers

\- format numbers and currency consistently

\- avoid overwhelming users with too many fields at once



5\. Design Principles

Inputs follow Ravlo’s core principles:

\- Blueprint alignment: strict grid structure

\- Cinematic spacing: generous breathing room

\- Clarity over cleverness: simple, readable fields

\- Emotional safety: calm, predictable interactions



Inputs are the touchpoints of the Ravlo OS. They transform user intent into structured data that powers analysis, workflows, and decision‑making.
