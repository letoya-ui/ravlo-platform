Buttons



Buttons are one of the most frequently used components in the Ravlo interface. They trigger actions, move users through workflows, and anchor key decisions across dashboards and OS modules.



Ravlo buttons are designed to feel:

\- confident

\- clear

\- intentional

\- cinematic

\- investor‑grade



1\. Purpose

Buttons communicate a single, clear action. They should always describe what will happen when the user clicks.



Examples:

\- Save

\- Continue

\- Add Property

\- Generate Report



2\. Types of Buttons

Ravlo uses a small, disciplined set of button types:



Primary Button

Used for the main action on a page or panel. High contrast, strong presence, and visually anchored.



Secondary Button

Used for supporting actions. Lower contrast but still clear and structured.



Tertiary Button

Used for light actions, filters, or non-critical interactions. Minimal styling.



Destructive Button

Used for irreversible or high‑risk actions. Always clear and never subtle.



3\. States

Every button must include the following states:

\- default

\- hover

\- pressed

\- disabled

\- loading



States help users understand what is interactive and what is not.



4\. Anatomy

A Ravlo button includes:

\- label text

\- optional icon

\- padding based on spacing tokens

\- radius based on radii tokens

\- shadow or glow based on elevation tokens



5\. Behavior

Buttons must:

\- use clear, action‑oriented labels

\- avoid vague text like “OK” or “Submit”

\- maintain consistent spacing and alignment

\- support keyboard and screen reader accessibility



6\. Design Principles

Buttons follow Ravlo’s core principles:

\- Blueprint alignment: consistent grid and spacing

\- Cinematic spacing: generous padding and breathing room

\- Clarity over cleverness: labels that communicate, not decorate

\- Emotional safety: stable, predictable interactions



Buttons are the heartbeat of interaction in the Ravlo ecosystem. They should always feel strong, intentional, and trustworthy.
