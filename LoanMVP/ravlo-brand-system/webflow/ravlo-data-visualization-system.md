# Ravlo Data Visualization System
The unified framework for charts, metrics, dashboards, and visual insights across the Ravlo OS.

---

# 1. Visualization Philosophy

Ravlo visualizations are:
- Clear  
- Calm  
- Investor-grade  
- Minimal and intentional  
- Designed to support decision-making  

Data should feel structured, not overwhelming.

---

# 2. Visualization Types

### 2.1 Charts
- Line charts  
- Bar charts  
- Donut charts  
- Area charts  
- Comparison charts  

### 2.2 Metrics
- KPI cards  
- Trend indicators  
- Delta values  

### 2.3 Tables
- Data tables  
- Sortable tables  
- Filterable tables  

### 2.4 Panels
- Summary panels  
- Insight panels  
- Breakdown panels  

---

# 3. Chart Anatomy

### Required Elements
- Title  
- Axis labels  
- Data lines/bars  
- Grid lines (subtle)  
- Tooltip  

### Optional Elements
- Legend  
- Trend line  
- Delta indicator  

---

# 4. Chart Tokens

### Line Weights
- line-thin: 1px  
- line-regular: 2px  

### Bar Radius
- radius-sm (4px)  

### Grid Lines
- 1px  
- 20ñ24px spacing  

### Motion
- 150ñ200ms ease-out  

---

# 5. Color Rules

### Primary Data Colors
- Blueprint  
- Steel  
- Graphite  

### Semantic Colors
- Success (green)  
- Warning (yellow)  
- Error (red)  

### Rules
- Never use more than 4 colors in a chart  
- Use semantic colors only when meaning is required  

---

# 6. KPI System

### KPI Anatomy
- Label  
- Value  
- Delta  
- Trend indicator  

### KPI Types
- Single metric  
- Metric with delta  
- Metric with sparkline  

### Rules
- Values must be large and clear  
- Deltas must use semantic colors  
- Sparklines must be subtle  

---

# 7. Dashboard Panels

### Panel Types
- Summary panel  
- Breakdown panel  
- Trend panel  
- Comparison panel  

### Panel Structure
- Title  
- Metric or chart  
- Supporting text  
- Optional CTA  

---

# 8. Table Visualization Rules

### Requirements
- Clear headers  
- Sort indicators  
- Hover states  
- Zebra striping (optional)  

### Accessibility
- aria-sort  
- Keyboard navigation  
- High contrast  

---

# 9. Tooltip System

### Tooltip Anatomy
- Label  
- Value  
- Timestamp (optional)  

### Rules
- No long text  
- No paragraphs  
- Must appear instantly  

---

# 10. Motion Guidelines

### Allowed
- Fade  
- Slide  
- Soft scale  

### Avoid
- Bounce  
- Overshoot  
- Excessive animation  

---

# 11. Data Density Rules

### Low Density
- Large spacing  
- Clear labels  
- Ideal for dashboards  

### Medium Density
- Compact spacing  
- More data points  

### High Density
- Only when necessary  
- Use tooltips to reduce clutter  

---

# 12. Visualization Principles

1. Clarity over complexity  
2. Calm over noise  
3. Structure over decoration  
4. Meaning over aesthetics  
5. Data must support decisions
