# Ravlo Iconography System
The engineered symbol language of the Ravlo OS — precise, architectural, and cinematic.

---

# 1. Iconography Philosophy

Ravlo icons are:
- Technical, not playful  
- Minimal, not decorative  
- Architectural, not cartoonish  
- Calm, not loud  
- Engineered, not ornamental  

Icons should feel like part of a premium operating system.

---

# 2. Icon Grid System

### Grid
- 24x24px base grid  
- 2px padding on all sides  
- Center-aligned strokes  

### Purpose
The grid ensures:
- Consistency  
- Balance  
- Predictable alignment  
- Clean scaling across devices  

---

# 3. Stroke Rules

### Stroke Weight
- **2px stroke** for all icons  
- Consistent across all sizes  

### Stroke Style
- Rounded caps  
- Rounded joins  
- No sharp corners unless intentional  

### Internal Spacing
- Minimum 2px between strokes  
- Avoid visual clutter  

---

# 4. Corner Radius Rules

### Rounded Corners
- 2–4px radius for shapes  
- Used for squares, rectangles, and UI metaphors  

### Circular Elements
- Perfect circles only  
- No ovals unless representing motion  

---

# 5. Icon Sizes

### Standard Sizes
- 16px — small UI  
- 20px — default UI  
- 24px — primary size  
- 32px — hero or panel headers  

### Rules
- Icons must scale proportionally  
- Never distort or stretch  
- Maintain stroke weight consistency  

---

# 6. Color Usage

### Default Icon Color
- Soft White (#F2F5F8) at 80–90% opacity  

### Secondary Icon Color
- Blueprint Cyan (#4CC3FF)  
- Used for:
  - Active states  
  - Blueprint moments  
  - Technical diagrams  

### Disabled Icons
- Soft White at 30–40% opacity

# 8. Motion Rules for Icons

### Hover
- Soft scale: 1.00 ? 1.05  
- Glow: subtle blueprint cyan  

### Active
- Scale: 0.98  
- Quick depress motion  

### Blueprint Icons
- Slow glow pulse (4–6s)  
- Optional drift (very subtle)  

---

# 9. Do & Don't Examples

### Do
- Keep icons minimal  
- Use consistent stroke weight  
- Maintain grid alignment  
- Use blueprint cyan sparingly  

### Don't
- Use filled icons  
- Mix stroke weights  
- Add unnecessary detail  
- Use playful or cartoonish styles  

---

# 10. Iconography Principles

1. Precision over decoration  
2. Engineering over illustration  
3. Calm over expressive  
4. Consistency builds trust  
5. Icons are part of the OS, not decoration
