# Ravlo Table System (Advanced)
The complete architecture for structured data presentation across the Ravlo OS — deals, properties, loans, partners, documents, and analytics.

---

# 1. Table Philosophy

Ravlo tables are:
- Clean and investor-grade  
- Highly scannable  
- Calm and predictable  
- Built for clarity and comparison  
- Designed to support decision-making  

Tables are not decorative — they are analytical tools.

---

# 2. Table Anatomy

### Required Elements
- Header row  
- Data rows  
- Sorting controls  
- Hover states  
- Active/selected states  
- Pagination or infinite scroll  
- Optional filters  

### Optional Elements
- Row actions  
- Inline tags  
- Status indicators  
- Icons  
- Expandable rows  

---

# 3. Table Variants

### Standard Table
- Default for most data  
- Clean, structured, minimal  

### Compact Table
- Reduced padding  
- Used for dense data  

### Data Grid
- Advanced filtering  
- Multi-column sorting  
- Sticky headers  
- Scrollable body  

### KPI Strip Table
- Horizontal metrics  
- Used for dashboards  

---

# 4. Column Types

### Text Column
- Left-aligned  
- Standard for names, descriptions  

### Numeric Column
- Right-aligned  
- Used for prices, values, metrics  

### Date Column
- Center or right-aligned  
- Consistent formatting  

### Status Column
- Uses semantic colors  
- Includes tag or pill  

### Icon Column
- Minimal  
- Used for quick scanning  

---

# 5. Row States

### Default
- Soft white text  
- Midnight background  

### Hover
- Slight highlight  
- Subtle glow  

### Active (Pressed)
- Light compress  
- Remove glow  

### Selected
- Blueprint border  
- Slight glow  

### Disabled
- Muted  
- No hover  

---

# 6. Sorting System

### Sorting Indicators
- Chevron up/down  
- Blueprint highlight when active  

### Sorting Rules
- One column sorted at a time (standard)  
- Multi-sort allowed in data grid  

### Sorting Behavior
- Smooth transition  
- No aggressive animation  

---

# 7. Pagination & Infinite Scroll

### Pagination
- Used for large datasets  
- Clear active page indicator  
- Next/previous controls  

### Infinite Scroll
- Used for dashboards  
- Must include loading indicator  
- Must preserve scroll position  

---

# 8. Filters & Controls

### Filter Types
- Dropdown filters  
- Tag filters  
- Range sliders  
- Search bar  

### Filter Behavior
- Smooth open/close  
- Clear focus state  
- Blueprint outline  

### Filter Placement
- Above table  
- Right-aligned or full-width  

---

# 9. Expandable Rows

### Use Cases
- Deal details  
- Property summaries  
- Document previews  

### Behavior
- Smooth slide-down  
- Blueprint border  
- Clear hierarchy  

---

# 10. Table Density Modes

### Comfortable
- space-16 row padding  
- Default for most pages  

### Compact
- space-8 row padding  
- Used for dense data  

### Spacious
- space-24 row padding  
- Used for investor summaries  

---

# 11. Accessibility Rules

- Rows must be keyboard navigable  
- Focus states must be visible  
- Sorting controls must be labeled  
- Status colors must meet contrast requirements  

---

# 12. Motion Guidelines

### Allowed
- Fade-in  
- Soft highlight  
- Smooth sorting transitions  

### Avoid
- Bouncy animations  
- Aggressive movement  

---

# 13. Table Principles

1. Clarity over decoration  
2. Predictability builds trust  
3. Every table must be scannable  
4. Motion must support comprehension  
5. Tables must feel like analytical tools
