# Ravlo Asset System
The complete architecture for Ravlo brand assets — logos, icons, patterns, exports, and delivery standards.

---

# 1. Asset Philosophy

Ravlo assets are:
- Cinematic and architectural  
- Clean and scalable  
- Engineered for clarity  
- Built for long-term consistency  
- Designed to work across OS, marketing, and partner surfaces  

Assets must feel premium, intentional, and unmistakably Ravlo.

# 3. Logo System

### Logo Variants
- ravlo-logo-primary  
- ravlo-logo-horizontal  
- ravlo-logo-icon  

### Clear Space
- 1x logo height on all sides  

### Minimum Sizes
- Digital: 24px height  
- Print: 0.5 inches  

### Color Rules
- Use blueprint on light backgrounds  
- Use white on dark backgrounds  
- Never apply gradients  

---

# 6. Export Specifications

### PNG
- 1x, 2x, 3x  
- Transparent background  

### SVG
- Clean paths  
- No embedded raster  
- No inline styles  

### WebP
- Used for Webflow  
- Optimized for performance  

### PDF
- Print-ready  
- CMYK  
- Vector only

### Logos

### Icons

- icon-[category]-[name].svg  

### Patterns
- pattern-blueprint-rings.svg  
- pattern-grid-overlay.svg  

### Exports
- ravlo-[asset]-[size].[format]

# 9. Usage Guidelines

### Do
- Use assets consistently  
- Maintain clear space  
- Use correct formats  
- Follow naming conventions  

### Don’t
- Stretch or distort  
- Change colors  
- Add effects  
- Combine patterns arbitrarily

# 10. Asset Principles

1. Assets must be clean and scalable  
2. Assets must reinforce the OS identity  
3. Assets must be easy to export and maintain  
4. Assets must feel cinematic and architectural
