# Ravlo Component States & Variants
A complete catalog of how Ravlo components adapt across sizes, states, and contextual variants.

---

# 1. Philosophy of Variants

Variants exist to:
- Maintain consistency  
- Reduce design drift  
- Support engineering reuse  
- Provide predictable behavior  
- Scale the OS across surfaces  

Every variant must feel like the same OS object — just adapted for context.

---

# 2. Buttons

## Sizes
- **Large** (44–52px height)  
- **Medium** (40–44px height)  
- **Small** (32–36px height)  

## Variants
- **Primary** (solid blueprint or soft white)  
- **Secondary** (outline)  
- **Ghost** (transparent)  
- **Icon Button** (square, minimal)  

## States
- Hover: soft scale + glow  
- Active: depress  
- Focus: blueprint outline  
- Disabled: 30–40% opacity

---

# 11. Variant Principles

1. Variants must reduce complexity, not add it  
2. States must be consistent across all components  
3. Every variant must feel like Ravlo  
4. Motion must support clarity  
5. Accessibility is required for every state
