# Ravlo Form System
The unified architecture for all inputs, fields, validation, and form interactions across the Ravlo OS.

---

# 1. Form Philosophy

Ravlo forms are:
- Calm and structured  
- Predictable and accessible  
- Designed to reduce cognitive load  
- Built for clarity and confidence  
- Consistent across all modules and workflows  

Forms are where decisions happen — they must feel trustworthy.

---

# 2. Form Anatomy

### Required Elements
- Label  
- Input field  
- Helper text (optional)  
- Error message (conditional)  
- Validation state  
- Icon (optional)  

### Layout
- Vertical stack  
- space-12 between label and input  
- space-8 between input and helper/error text  
- space-24 between fields  

---

# 3. Input Types

### Text Input
- Standard text  
- Email  
- Phone  
- Number  
- Password  

### Textarea
- Multi-line input  
- Auto-resize optional  

### Select
- Dropdown  
- Searchable select (optional)  

### Checkbox
- Single checkbox  
- Checkbox group  

### Radio
- Radio group  
- Horizontal or vertical layout  

### Toggle
- Binary on/off  

### Slider
- Range selection  

### Date Picker
- Calendar dropdown  

---

# 4. Input States

### Default
- Soft white text  
- Midnight border  
- Subtle shadow  

### Hover
- Border highlight  
- Slight glow  

### Focus
- Blueprint outline  
- Soft glow  
- Clear emphasis  

### Active
- Crisp feedback  

### Error
- Red border  
- Red helper text  
- Icon optional  

### Disabled
- 30–40% opacity  
- No hover or focus  

---

# 5. Validation System

### Validation Types
- Required  
- Email format  
- Phone format  
- Number range  
- Character limits  
- Custom rules  

### Error Messaging Rules
- Clear and concise  
- One error at a time  
- No technical jargon  
- Must appear immediately after validation fails  

---

# 6. Form Layout Patterns

### Single Column Form
- Standard for onboarding  
- Clear vertical rhythm  

### Two Column Form
- For advanced investor workflows  
- Use ravlo-grid-2  

### Multi-Step Form
- Progress indicator  
- Save state between steps  
- Clear CTA placement  

### Inline Form
- Search bars  
- Quick filters  

---

# 7. CTA Placement

### Primary CTA
- Bottom-right of form  
- Full-width on mobile  

### Secondary CTA
- Bottom-left or below primary  

### Disabled CTA
- Only when validation fails  

---

# 8. Accessibility Rules

- Labels must always be visible  
- Placeholder is not a label  
- Focus states must be clear  
- Error messages must be announced  
- Inputs must be keyboard navigable  

---

# 9. Motion & Interaction

### Allowed Motion
- Soft fade-in  
- Subtle border glow  
- Smooth dropdown open/close  

### Avoid
- Bouncy animations  
- Aggressive movement  
- Parallax  

---

# 10. Form Principles

1. Clarity over complexity  
2. Predictability builds trust  
3. Accessibility is mandatory  
4. Motion must support comprehension  
5. Forms must reduce cognitive load
