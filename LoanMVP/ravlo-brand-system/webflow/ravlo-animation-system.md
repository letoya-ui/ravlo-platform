# Ravlo Animation System
Calm, cinematic motion rules for the Ravlo Visual OS.

---

# 1. Motion Philosophy

Ravlo motion is:
- Calm, not hyperactive  
- Cinematic, not playful  
- Intentional, not decorative  

Animations should feel like the OS is breathing ó never distracting from investor decisions.

---

# 2. Core Motion Types

### Fade In
- Duration: 200ñ400ms  
- Easing: ease-out  
- Usage: sections, cards, panels entering view  

### Slide Up
- Distance: 20ñ40px  
- Duration: 250ñ400ms  
- Easing: ease-out  
- Usage: hero content, panel bodies, key text blocks  

### Soft Scale
- Scale: 1.00 ? 1.02  
- Duration: 150ñ250ms  
- Easing: ease-out  
- Usage: hover on cards, buttons, key CTAs  

### Glow Pulse
- Opacity: subtle (e.g., 0.4 ? 0.6)  
- Duration: 4ñ6s loop  
- Usage: blueprint rings, key OS nodes  

### Blueprint Drift
- Movement: 5ñ15px slow drift  
- Duration: 20ñ30s loop  
- Usage: background blueprint geometry  

---

# 3. Component Motion Rules

### Panels
- Entry: fade-in + slight slide-up  
- Hover: very subtle lift (2ñ4px) + soft shadow increase  
- No rotation, no aggressive scaling  

### Cards
- Hover: soft scale (1ñ2%) + slight shadow deepen  
- Entry: fade-in staggered in grids  

### Hero
- Content: fade-in + slide-up  
- Background: slow blueprint drift + soft glow pulse  

---

# 4. Timing & Easing

### Durations
- Micro: 120ñ180ms (small UI feedback)  
- Standard: 200ñ400ms (sections, cards, panels)  
- Macro: 600ñ1000ms (hero, large transitions)  

### Easing
- Default: ease-out  
- For subtle loops: linear  

---

# 5. Staggered Motion

For grids and lists:
- Stagger delay: 40ñ80ms per item  
- Max stagger: 6ñ8 items  
- Usage: feature grids, pricing cards, module lists  

Stagger should feel like a calm reveal, not a cascade.

---

# 6. Scroll Behavior

### On Scroll
- Sections: fade-in + slide-up when entering viewport  
- Panels: fade-in only, minimal movement  
- Cards: staggered fade-in in grids  

Avoid parallax-heavy behavior ó keep motion grounded and intentional.

---

# 7. Interaction Feedback

### Buttons
- Hover: soft scale + slight glow  
- Active: quick depress (scale 0.98)  

### Inputs
- Focus: subtle glow + border highlight  

Feedback should feel precise and investor-grade.

---

# 8. Motion Do's and Don'ts

### Do
- Use motion to guide attention  
- Keep transitions consistent  
- Maintain calm, OS-like behavior  

### Don't
- Use bouncy or elastic easing  
- Overuse large movements  
- Animate purely for decoration  

---

# 9. Motion Principles

1. Motion supports decisions, not distracts from them  
2. The OS feels alive, not noisy  
3. Every animation has a purpose  
4. Subtlety is the default  
5. Consistency builds trust
