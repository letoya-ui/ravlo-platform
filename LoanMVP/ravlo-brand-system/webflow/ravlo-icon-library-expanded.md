# Ravlo Icon Library (Expanded)
The complete iconography system for the Ravlo OS ó structure, categories, stroke rules, and usage patterns.

---

# 1. Icon Philosophy

Ravlo icons are:
- Architectural and geometric  
- Minimal and precise  
- Built on a strict grid  
- Designed for clarity and scannability  
- Consistent across all modules and surfaces  

Icons should feel engineered, not decorative.

---

# 2. Icon Grid System

### Grid
- 24x24 base grid  
- 2px padding  
- 20x20 active drawing area  

### Stroke Rules
- Stroke weight: 1.5px  
- Rounded caps  
- Rounded joins  

### Corner Radius
- Subtle rounding for internal shapes  
- No sharp 90∞ corners unless intentional  

---

# 3. Icon Sizes

### Standard Sizes
- 16px  
- 20px  
- 24px (default)  
- 32px (large panels)  

### Scaling Rules
- Icons must scale proportionally  
- Stroke weight remains consistent

# 5. Icon Construction Rules

### Geometry
- Use simple shapes  
- Avoid unnecessary detail  
- Maintain symmetry  

### Alignment
- Center icons within grid  
- Align strokes to pixel grid  

### Negative Space
- Maintain consistent internal spacing  
- Avoid clutter  

---

# 6. Icon States

### Default
- Soft white or blueprint  
- No glow  

### Hover
- Slight glow  
- Subtle scale (1.02)  

### Active
- Quick depress  
- Remove glow  

### Disabled
- 30ñ40% opacity

# 7. Semantic Color Rules

### Success
- Green token  

### Warning
- Yellow token  

### Error
- Red token  

### Info
- Blueprint token  

### Neutral
- Soft white or steel blue  

---

# 8. Usage Guidelines

### Do
- Use icons to support clarity  
- Pair icons with labels when needed  
- Keep icons consistent across modules  

### Donít
- Use icons as decoration  
- Mix icon styles  
- Overuse icons in dense layouts  

---

# 9. Accessibility Rules

- Icons must have accessible labels  
- Icons must meet contrast requirements  
- Icons must not be the only indicator of meaning  

---

# 10. Icon Principles

1. Icons must feel engineered  
2. Geometry must be consistent  
3. Icons must support clarity
