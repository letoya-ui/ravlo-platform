# \# Quick Actions

# 

# Quick Actions provide fast access to high‑value tasks such as adding a deal, running analysis, adding a property, or contacting a lender. They are designed for speed and focus.

# 

# \## Purpose

# 

# Quick Actions answer:  

# “What can I do right now that moves my portfolio forward?”

# 

# They should:

# \- surface the most common and impactful actions  

# \- reduce friction for starting workflows  

# \- feel predictable and consistent across sessions  

# 

# \## Example Actions

# 

# \- Add a new deal  

# \- Run deal analysis  

# \- Add a property to portfolio  

# \- Contact a lender  

# \- Create a task or reminder  

# \- Upload documents  

# 

# \## Anatomy

# 

# Each Quick Action typically includes:

# 

# \- Icon  

# \- Label (short, action‑oriented)  

# \- Optional: short description or tooltip  

# 

# \## Layout

# 

# \- Displayed as a row or grid of buttons or tiles.  

# \- Positioned near the top or middle of the Command Center, where it’s easy to reach.  

# \- Uses consistent sizing and spacing.  

# 

# \## Component References

# 

# \- Primary and secondary buttons  

# \- Icon buttons or tiles  

# \- Tooltips  

# \- Optional: dropdown for additional actions  

# 

# \## Behavior

# 

# \- Actions should open the relevant module or modal (e.g., Deal Architect, Capital Studio, Tasks).  

# \- Some actions may open pre‑filled forms based on current context (e.g., from a selected property).  

# \- Quick Actions should be keyboard accessible and screen‑reader friendly.  

# 

# \## States

# 

# \- \*\*Default:\*\* All actions available.  

# \- \*\*Disabled:\*\* Certain actions may be disabled if prerequisites are not met (e.g., no lenders connected).  

# \- \*\*Empty:\*\* For new users, show a smaller set of guided actions (e.g., “Connect a lender,” “Add your first property”).  

# 

# \## Do

# 

# \- Keep labels short and clear (e.g., “Add Deal,” not “Initiate Deal Creation Workflow”).  

# \- Prioritize actions that investors use frequently.  

# \- Ensure actions feel responsive and immediate.  

# 

# \## Don’t

# 

# \- Overload the Quick Actions area with rarely used options.  

# \- Use vague labels that don’t clearly indicate what will happen.  

# \- Hide critical actions behind multiple nested menus.
