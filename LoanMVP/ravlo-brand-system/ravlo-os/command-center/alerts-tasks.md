# \# Alerts \& Tasks

# 

# Alerts \& Tasks highlight deadlines, overdue items, lender requests, and risk‑related notifications. This section ensures investors never lose sight of what needs action.

# 

# \## Purpose

# 

# Alerts \& Tasks answer:  

# “What requires my attention right now?”

# 

# They should:

# \- surface time‑sensitive and risk‑sensitive items  

# \- make it easy to act or triage  

# \- reduce the chance of missed deadlines or issues  

# 

# \## Types

# 

# \- Overdue tasks  

# \- Upcoming deadlines (e.g., closing dates, rate locks, inspections)  

# \- Lender or partner requests  

# \- Risk alerts (e.g., negative cash flow, high vacancy, expiring insurance)  

# 

# \## Anatomy

# 

# Each alert or task includes:

# 

# \- Title (short, action‑oriented or descriptive)  

# \- Type (task, alert, request)  

# \- Due date or urgency indicator  

# \- Related entity (deal, property, lender, portfolio)  

# \- Primary action (e.g., “View,” “Complete,” “Respond”)  

# 

# \## Layout

# 

# \- Displayed as a list or grouped by urgency (e.g., “Overdue,” “Due soon,” “Upcoming”).  

# \- Positioned where it’s easy to see without overwhelming the page.  

# 

# \## Component References

# 

# \- List / task item  

# \- Badges for urgency and type  

# \- Checkboxes or completion controls (for tasks)  

# \- Buttons or links for actions  

# 

# \## Behavior

# 

# \- Tasks can be marked complete directly from the Command Center.  

# \- Alerts should deep‑link into the relevant module or detail view.  

# \- Items may be snoozed or dismissed where appropriate, with clear rules.  

# 

# \## States

# 

# \- \*\*Normal:\*\* A manageable list of tasks and alerts.  

# \- \*\*High urgency:\*\* Multiple overdue or critical items; visual hierarchy should emphasize the most important.  

# \- \*\*Empty:\*\* If there are no alerts or tasks, show a calm message (e.g., “You’re all caught up.”).  

# \- \*\*Error:\*\* If alerts or tasks cannot load, show a clear fallback state.  

# 

# \## Do

# 

# \- Prioritize clarity and urgency without creating panic.  

# \- Make it easy to complete or address items quickly.  

# \- Keep labels and actions specific and understandable.  

# 

# \## Don’t

# 

# \- Mix low‑priority notifications with critical alerts without distinction.  

# \- Require multiple clicks for simple completions.  

# \- Use ambiguous urgency indicators.
