# \# Portfolio Metrics

# 

# Portfolio Metrics display key investor KPIs at a glance, including equity, cash flow, ROI, and risk indicators. They are designed for clarity, legibility, and quick scanning.

# 

# \## Purpose

# 

# Portfolio Metrics answer:  

# “How is my portfolio performing right now?”

# 

# They should:

# \- surface the most important financial indicators  

# \- show trends over time  

# \- highlight risk or anomalies  

# 

# \## Core Metrics

# 

# Recommended default metrics:

# 

# \- Total portfolio value  

# \- Total equity  

# \- Monthly cash flow  

# \- Portfolio ROI  

# \- Occupancy / utilization (if applicable)  

# \- Risk / exposure indicator  

# 

# These can be adapted based on investor type and configuration.

# 

# \## Anatomy

# 

# Each metric card typically includes:

# 

# \- Label (e.g., “Total Equity”)  

# \- Primary value (e.g., “$1.2M”)  

# \- Trend indicator (e.g., “+4.2% vs last 30 days”)  

# \- Optional: status or risk badge  

# 

# \## Layout

# 

# \- Displayed in a row or grid near the top of the Command Center.  

# \- Use consistent sizing and spacing between metric cards.  

# \- Ensure values are legible at a glance, with clear typographic hierarchy.  

# 

# \## Component References

# 

# \- Metric card  

# \- Badge / status indicator  

# \- Sparkline or mini chart (optional)  

# \- Tooltip (for definitions and calculation details)  

# 

# \## Behavior

# 

# \- Metrics should update in near real‑time or on a predictable refresh cadence.  

# \- Hover or tap can reveal more detail (e.g., how a metric is calculated).  

# \- Clicking a metric should deep‑link into Portfolio Intelligence or a relevant detail view.  

# 

# \## States

# 

# \- \*\*Normal:\*\* All metrics available and within expected ranges.  

# \- \*\*Warning:\*\* One or more metrics flagged (e.g., negative cash flow, high vacancy).  

# \- \*\*Error:\*\* If data cannot load, show a clear fallback state per metric.  

# \- \*\*Empty:\*\* For new users, show placeholders and brief explanations of what will appear once data is connected.  

# 

# \## Do

# 

# \- Prioritize clarity over density.  

# \- Use consistent formatting for currency, percentages, and time ranges.  

# \- Make it obvious when a metric is trending positively or negatively.  

# 

# \## Don’t

# 

# \- Mix too many different visual styles in one metric row.  

# \- Use ambiguous labels or unexplained abbreviations.  

# \- Overwhelm the investor with more than 6–8 metrics on the main view.
