# \# Recent Activity

# 

# Recent Activity shows a timeline of updates across deals, tasks, messages, and partner actions. It keeps investors aware of what has changed since their last session.

# 

# \## Purpose

# 

# Recent Activity answers:  

# “What happened while I was away?”

# 

# It should:

# \- show a chronological feed of important events  

# \- group related actions where possible  

# \- make it easy to drill into specific items  

# 

# \## Activity Types

# 

# \- Deal updates (status changes, new notes, new documents)  

# \- Task updates (created, completed, overdue)  

# \- Lender or partner actions  

# \- Portfolio changes (new properties, updated values)  

# \- System events (e.g., AI analysis completed)  

# 

# \## Anatomy

# 

# Each activity item includes:

# 

# \- Actor or source (e.g., “You,” “Lender,” “Ravlo AI”)  

# \- Description (short, clear)  

# \- Timestamp  

# \- Optional: related entity (deal, property, task)  

# \- Optional: status or type badge  

# 

# \## Layout

# 

# \- Vertical timeline or list.  

# \- Most recent items at the top.  

# \- Grouped by day where appropriate.  

# 

# \## Component References

# 

# \- List / timeline component  

# \- Avatars or icons for actors  

# \- Badges for activity type  

# \- Links to related entities  

# 

# \## Behavior

# 

# \- Clicking an item should open the related deal, task, or module.  

# \- Activity should update in near real‑time or on refresh.  

# \- Older activity may be collapsed behind “View more” to keep the view focused.  

# 

# \## States

# 

# \- \*\*Normal:\*\* A mix of recent events across deals and tasks.  

# \- \*\*High activity:\*\* Many events; grouping and batching become more important.  

# \- \*\*Empty:\*\* For new users, show an explanation and encourage first actions (e.g., “Add a deal to start seeing activity here.”).  

# \- \*\*Error:\*\* If activity cannot load, show a clear message and retry option.  

# 

# \## Do

# 

# \- Keep descriptions short and specific.  

# \- Use consistent timestamps and formatting.  

# \- Make it obvious what each activity relates to.  

# 

# \## Don’t

# 

# \- Show low‑value noise that clutters the feed.  

# \- Use vague descriptions like “Something changed.”  

# \- Bury important events deep in the list without highlighting them elsewhere.
