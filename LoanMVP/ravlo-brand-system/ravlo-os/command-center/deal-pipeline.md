# \# Deal Pipeline

# 

# The Deal Pipeline visualizes the lifecycle of deals across stages such as New, Active, Underwriting, Rehab, On Market, and Closed. It helps investors understand where attention is needed.

# 

# \## Purpose

# 

# The Deal Pipeline answers:  

# “What’s happening with my deals, and where are they stuck?”

# 

# It should:

# \- show the count and value of deals in each stage  

# \- highlight bottlenecks and risks  

# \- provide quick access to specific deals  

# 

# \## Stages

# 

# Typical stages may include:

# 

# \- New / Leads  

# \- Active / In Review  

# \- Underwriting  

# \- Rehab / In Progress  

# \- On Market / Disposition  

# \- Closed  

# 

# Stages should be configurable per investor or strategy where appropriate.

# 

# \## Anatomy

# 

# Each stage column or segment includes:

# 

# \- Stage name  

# \- Count of deals  

# \- Optional: total projected value or equity  

# \- Optional: risk or attention indicator  

# 

# \## Layout

# 

# \- Horizontal pipeline or column‑based layout.  

# \- Each stage visually distinct but stylistically consistent.  

# \- Clickable segments that open filtered views of deals in that stage.  

# 

# \## Component References

# 

# \- Pipeline / segmented layout  

# \- Cards or list items for deals (in drill‑down views)  

# \- Badges and status indicators  

# \- Filters and sorting controls  

# 

# \## Behavior

# 

# \- Clicking a stage opens a filtered list of deals in that stage.  

# \- Hover or tap can show a quick summary (e.g., “3 deals, $850K projected value”).  

# \- Risk or attention flags should be visually distinct but not chaotic.  

# 

# \## States

# 

# \- \*\*Normal:\*\* Deals distributed across stages.  

# \- \*\*High load:\*\* Many deals in one or more stages; layout should remain readable.  

# \- \*\*Empty:\*\* For new users, show an explanation and a CTA to “Add your first deal” or “Open Deal Architect.”  

# \- \*\*Error:\*\* If pipeline data cannot load, show a clear message and retry option.  

# 

# \## Do

# 

# \- Keep stage names clear and aligned with investor workflows.  

# \- Make it easy to see where deals are piling up.  

# \- Ensure the pipeline feels like a navigational element, not just a static chart.  

# 

# \## Don’t

# 

# \- Overcomplicate the pipeline with too many stages on the main view.  

# \- Hide critical risk indicators behind multiple clicks.  

# \- Use inconsistent terminology between the pipeline and Deal Architect.
