# \# Architectural Styles

# 

# Architectural Styles define the visual identity of a project. They influence materials, presets, cost ranges, and AI generation outputs.

# 

# \## Purpose

# 

# This step answers:

# “What architectural language should this project speak?”

# 

# It provides:

# \- style definitions  

# \- visual references  

# \- material compatibility  

# \- cost implications  

# \- AI generation context  

# 

# \## Example Styles

# 

# \- Modern  

# \- Contemporary  

# \- Craftsman  

# \- Farmhouse  

# \- Mid‑Century  

# \- Minimalist  

# \- Industrial  

# 

# \## Anatomy

# 

# Each style includes:

# \- Name  

# \- Description  

# \- Visual reference  

# \- Material palette  

# \- Typical cost range  

# \- AI generation tags  

# 

# \## Behavior

# 

# \- Selecting a style updates available materials and presets.  

# \- AI generation uses style metadata to shape renders and blueprints.  

# 

# \## Do

# 

# \- Keep style descriptions short and evocative.  

# \- Use consistent visual references.  

# 

# \## Don’t

# 

# \- Mix incompatible styles without explanation.  

# \- Overwhelm users with niche or hyper‑specific styles.
