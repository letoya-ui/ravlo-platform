# Ravlo Discovery Loop

## The Product Pattern Behind “What Happens Next”

Ravlo should never feel like a dead end.

Every screen, module, and workflow should naturally answer the user's next question:

> What happens next?

This is the difference between feature overload and a true operating system.

A user should not feel like Ravlo is throwing every tool at them at once. They should feel like each action unlocks the next helpful step in their real estate journey.

---

## Core Principle

Ravlo is a rabbit hole in a good way.

Not because users are lost.

Because every next step makes sense.

The experience should feel like guided discovery:

1. The user comes in for one problem.
2. Ravlo helps them solve it.
3. Ravlo reveals the next logical tool.
4. The user goes deeper because the journey feels useful, not overwhelming.

---

## Investor Discovery Loop

A new or active investor should naturally move through this path:

```text
Learn
  ↓
Find Opportunities
  ↓
Analyze Deals
  ↓
Design / Scope the Project
  ↓
Build the Budget
  ↓
Track Costs
  ↓
Secure Financing
  ↓
Grow the Portfolio / Business
```

This loop supports the Ravlo promise:

> Helping people make smarter real estate decisions — from their first investment property to running an entire real estate business.

---

## Dashboard Pattern

The Investor Dashboard already supports this pattern through:

- **Find** — search a market and save promising opportunities.
- **Underwrite** — compare strategy, score, ARV, rehab, and rent assumptions.
- **Plan** — move the right deal into scope, budget, build, or design.
- **Fund** — submit complete deals into the capital workflow.
- **Next Best Move** — tell the user what to do now.

This pattern should be protected and expanded across Ravlo.

---

## Every Module Should Answer

Each Ravlo module should answer these questions:

### 1. Where am I?

The user should immediately understand what part of the journey they are in.

Examples:

- Learning
- Sourcing
- Underwriting
- Planning
- Budgeting
- Funding
- Managing
- Growing

### 2. What can I do here?

The screen should make the primary action clear.

Examples:

- Analyze this deal
- Save this property
- Generate a renovation budget
- Send to Design Studio
- Request capital
- Complete the next lesson

### 3. What happens next?

The screen should always suggest the next logical action.

Examples:

- After saving a property, open Deal Workspace.
- After analyzing a deal, open Deal Architect.
- After Deal Architect, send it to Budget Studio.
- After Budget Studio, track expenses or request capital.
- After Academy, apply the lesson to a real property.

---

## Ravlo AI Role

Ravlo AI should not only answer questions.

It should guide next steps.

Good AI behavior:

- "This property is worth investigating. Open Deal Architect to compare strategies."
- "Your rehab estimate is missing exterior scope. Add it before requesting capital."
- "You completed the financing lesson. Apply it to one saved property."
- "Your deal looks close to funding-ready. Review the missing documents."

Bad AI behavior:

- Generic advice.
- Ending with no action.
- Explaining features without moving the user forward.

---

## Product Rule

Before adding or changing any feature, ask:

> Does this help the user know what to do next?

If the answer is no, the feature needs a clearer purpose, better placement, or should not be built yet.

---

## Why This Matters

Ravlo is not just a collection of tools.

Ravlo is an operating system for the real estate journey.

The product becomes powerful when each tool connects to the next one, and the user feels supported from the first question to the final outcome.

The goal is not to make users explore everything.

The goal is to make the next step feel obvious.
