# \# Detail View Patterns

# 

# Detail Views provide deep, structured information about a single entity — a deal, property, partner, or loan.

# 

# \## Purpose

# 

# Detail Views answer:

# “What is the full story of this item?”

# 

# They provide:

# \- hero section  

# \- summary metrics  

# \- tabs or sections  

# \- data panels  

# \- activity  

# \- documents  

# 

# \## Anatomy

# 

# \- Header  

# \- Summary metrics  

# \- Tab bar  

# \- Panels  

# \- Activity feed  

# \- Documents section  

# 

# \## Layout Rules

# 

# \- Use a two‑column layout where possible.  

# \- Keep summary metrics visible.  

# \- Use tabs for deep content.  

# 

# \## Do

# 

# \- Keep detail views structured and calm.  

# \- Use consistent panel spacing.  

# 

# \## Don’t

# 

# \- Mix unrelated content in one panel.
