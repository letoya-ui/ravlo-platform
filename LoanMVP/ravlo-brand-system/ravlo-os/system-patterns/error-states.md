# \# Error State Patterns

# 

# Error States help users recover from issues gracefully.

# 

# \## Purpose

# 

# Error States answer:

# “What went wrong, and how do I fix it?”

# 

# They provide:

# \- clear messaging  

# \- retry actions  

# \- fallback content  

# 

# \## Types

# 

# \- Network error  

# \- Data load error  

# \- Permission error  

# \- System error  

# 

# \## Do

# 

# \- Keep messages calm and clear.  

# \- Provide a retry action.  

# 

# \## Don’t

# 

# \- Use technical jargon.

#
