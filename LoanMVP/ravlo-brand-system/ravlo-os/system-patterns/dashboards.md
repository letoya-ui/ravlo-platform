# \# Dashboard Patterns

# 

# Dashboard Patterns define the structure of high‑level overview screens across Ravlo OS.

# 

# \## Purpose

# 

# Dashboards answer:

# “What’s happening at a glance?”

# 

# They provide:

# \- hero sections  

# \- metric rows  

# \- card grids  

# \- panels  

# \- activity feeds  

# \- AI insights  

# 

# \## Core Dashboard Types

# 

# \- Command Center Dashboard  

# \- Deal Dashboard  

# \- Property Dashboard  

# \- Portfolio Dashboard  

# \- Partner Dashboard  

# 

# \## Anatomy

# 

# \- Header  

# \- Hero panel  

# \- Metric row  

# \- Card grid  

# \- Panels  

# \- Activity feed  

# \- AI insights  

# 

# \## Layout Rules

# 

# \- Use a 12‑column grid.  

# \- Keep metrics at the top.  

# \- Use breathing room between sections.  

# 

# \## Do

# 

# \- Keep dashboards scannable.  

# \- Use consistent hierarchy.  

# 

# \## Don’t

# 

# \- Overload dashboards with dense data.
