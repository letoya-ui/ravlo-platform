# \# Empty State Patterns

# 

# Empty States guide users when no data exists yet.

# 

# \## Purpose

# 

# Empty States answer:

# “What should I do next?”

# 

# They provide:

# \- explanation  

# \- illustration or icon  

# \- primary action  

# 

# \## Types

# 

# \- First‑time user  

# \- No results  

# \- No activity  

# \- No documents  

# 

# \## Do

# 

# \- Keep empty states friendly and helpful.  

# \- Provide a clear next step.  

# 

# \## Don’t

# 

# \- Leave screens blank.
