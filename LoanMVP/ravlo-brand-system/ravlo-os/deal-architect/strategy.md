# \# Strategy Selection

# 

# Strategy Selection defines the investment approach for the deal. It determines the calculations, assumptions, and workflows used throughout Deal Architect.

# 

# \## Purpose

# 

# This step answers:  

# “What type of deal is this?”

# 

# It ensures:

# \- correct formulas  

# \- correct valuation models  

# \- correct rehab and financing assumptions  

# \- correct AI analysis context  

# 

# \## Supported Strategies

# 

# \- Fix \& Flip  

# \- Rental  

# \- BRRRR  

# \- Airbnb / Short‑Term Rental  

# \- New Construction (optional future module)  

# 

# \## Anatomy

# 

# \- Strategy cards  

# \- Short descriptions  

# \- Expected timeline  

# \- Expected capital requirements  

# \- Risk profile indicators  

# 

# \## Behavior

# 

# \- Selecting a strategy updates the workflow dynamically.  

# \- Strategy can be changed until the deal is finalized.  

# 

# \## Do

# 

# \- Keep strategy descriptions simple and clear.  

# \- Show differences between strategies visually.  

# 

# \## Don’t

# 

# \- Force users to commit without explanation.  

# \- Hide strategy‑specific assumptions.
