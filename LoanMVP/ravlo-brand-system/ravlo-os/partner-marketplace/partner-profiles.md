# \# Partner Profiles

# 

# Partner Profiles provide detailed information about each partner, including services, experience, and portfolio.

# 

# \## Purpose

# 

# This step answers:

# “Is this the right partner for my deal?”

# 

# It provides:

# \- bio  

# \- services  

# \- coverage areas  

# \- portfolio photos  

# \- reviews  

# \- pricing (optional)  

# \- service request CTA  

# 

# \## Anatomy

# 

# \- Header (name, role, rating)  

# \- Bio  

# \- Services list  

# \- Coverage map  

# \- Portfolio gallery  

# \- Reviews  

# \- Contact / request button  

# 

# \## Behavior

# 

# \- Public users see a limited profile.  

# \- Logged‑in users see full details.  

# \- Partners can update their own profiles.  

# 

# \## Do

# 

# \- Keep profiles visually rich.  

# \- Highlight services clearly.  

# 

# \## Don’t

# 

# \- Overload profiles with unnecessary fields.
