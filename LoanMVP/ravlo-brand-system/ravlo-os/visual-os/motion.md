# \# Motion \& Transitions

# 

# Motion defines how Ravlo OS feels when it moves — calm, intentional, and premium.

# 

# \## Purpose

# 

# This step answers:

# “How does Ravlo move?”

# 

# It provides:

# \- transition rules  

# \- easing curves  

# \- panel motion  

# \- micro‑interactions  

# 

# \## Motion Principles

# 

# \- Slow in, slow out  

# \- Ease‑in‑out curves  

# \- 150–300ms transitions  

# \- No bounce or elastic motion  

# 

# \## Do

# 

# \- Keep motion subtle and purposeful.  

# \- Use motion to reinforce hierarchy.  

# 

# \## Don’t

# 

# \- Use playful or bouncy animations.  

# \- Over‑animate simple interactions.
